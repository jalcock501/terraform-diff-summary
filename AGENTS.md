# AGENTS.md

## Project Overview

This repository contains `terraform-diff-summary`, a composite GitHub Action
that reads Terraform plan JSON and writes a compact Markdown summary to the
GitHub Step Summary.

The action is intentionally small:

- The action entrypoint is `scripts/terraform_diff_summary.py`.
- Runtime modules live in `scripts/terraform_plan_summary/`.
- Tests live in `tests/test_terraform_diff_summary.py`.
- Action metadata lives in `action.yml`.
- CI workflows live in `.github/workflows/`.

## Core Behavior

- Read Terraform JSON generated by `terraform show -json tfplan`.
- Ignore Terraform `no-op` resource changes.
- Optionally filter updates where the only effective changes are configured
  tag keys under `tags` or `tags_all`.
- Render changed field paths only, not before/after values.
- Append Markdown to `$GITHUB_STEP_SUMMARY`.
- Keep Python runtime dependency-free and standard-library only.

## Development Commands

Install dependencies:

```bash
poetry install
```

Run tests:

```bash
poetry run test
```

Run lint checks:

```bash
poetry run ruff check .
```

## Coding Guidelines

- Keep changes small, explicit, and task-scoped.
- Prefer existing function names and patterns unless there is a clear reason to
  change them.
- Preserve backward compatibility for existing action inputs where practical.
- Do not add runtime dependencies.
- Use standard-library Python for action behavior.
- Show changed field paths, never Terraform before/after values.
- Avoid broad refactors unless they directly support the requested change.
- Update README examples and input docs when action behavior changes.

## Testing Guidelines

- Add or update pytest coverage for meaningful behavior changes.
- Cover both the rendered Markdown and lower-level helper behavior when changing
  diff/filter logic.
- Keep tests synthetic. Do not use real cloud account names, secrets, tokens, or
  production Terraform output.
- Run `poetry run test` and `poetry run ruff check .` before finishing.

## GitHub Actions Guidelines

- The composite action should continue to call:

  ```bash
  python "$GITHUB_ACTION_PATH/scripts/terraform_diff_summary.py"
  ```

- CI should test both Python logic and the real local action via `uses: ./`.
- Tagging/release workflow changes should be conservative and avoid rewriting
  existing tags.

## Security and Privacy

- Treat Terraform plan data as potentially sensitive.
- Do not add output that prints resource values from `before`, `after`, or
  `after_unknown`.
- Do not include secrets or realistic production identifiers in tests, docs, or
  examples.
- Ask before adding dependencies, network calls, destructive git commands, or
  infrastructure commands.

## Review Priorities

When reviewing changes, focus on:

- Incorrect filtering of material Terraform changes.
- Accidentally hiding visible tag-only updates when filtering is disabled.
- Leaking before/after values into summaries.
- Broken action input wiring between `action.yml` and environment variables.
- Missing tests for new filtering or rendering behavior.
